## üìã **ADJUSTMENT PROTOCOL**

### **When Adjustments Are Allowed**
- System crashes or freezes during test execution
- API endpoint failures preventing test completion
- Database connectivity issues
- Resource exhaustion (memory, CPU)
- Configuration errors discovered during testing

### **Adjustment Process**
1. **Pause Trial:** Immediately stop current test execution
2. **Document Issue:** Log the problem with timestamps and evidence
3. **Repair:** Apply minimal fix to restore functionality
4. **Restart:** Resume from the exact point of failure (if possible) or restart test
5. **Log Adjustment:** Document all changes in this audit trail
6. **Resume Trial:** Continue with next test in sequence

### **What Gets Logged**
- Timestamp of issue detection
- Test being executed when issue occurred
- Detailed description of the problem
- Root cause analysis (if determined)
- Exact repair actions taken
- System state before/after repair
- Impact on trial timeline
- Moderator approval for adjustment
